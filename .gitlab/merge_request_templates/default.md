## 自查清单

请在提交合并请求之前，确保您已经完成以下检查：

### 代码质量
- [ ] 代码是否经过了自我审查？
- [ ] 是否每个提交都使用了pre-commit功能？
- [ ] 代码是否符合项目的编码规范？
- [ ] 是否删除了所有调试信息（如 `console.log`、`printk`信息 等）？
- [ ] 是否在代码中添加了必要的注释？
- [ ] 提交信息是否完全说明了此次提交的修改情况？
- [ ] 是否有遗漏的文件没有提交？
- [ ] 是否每个代码都添加了版权说明？
- [ ] 关联子模块是否更新？

### 功能测试
- [ ] 新增或修改的功能是否经过了充分的测试？
- [ ] 是否充分考虑了不同平台的影响域？

### 文档更新
- [ ] 是否更新了相关的文档（如接口文档、README 等）？

### 依赖管理
- [ ] 是否检查并更新了依赖项？
- [ ] 是否移除了不再使用的依赖项？
- [ ] 是否配置了默认的依赖项？

### 其他
- [ ] 是否考虑了代码的性能和优化？
- [ ] 是否处理了所有相关的代码审查意见？

请确保以上所有项都已完成，以便更快地通过代码审查。感谢您的合作！
