# 中文 Commit 规范

## 标题格式

优先使用下面两种格式之一：

- `类型: 动作结果`
- `模块: 动作结果`

标题要短、准、可回看，避免“更新”“修改一下”“处理问题”这类空泛表达。

## 常用类型

- `新增`: 引入新能力
- `修复`: 修正缺陷或兼容性问题
- `重构`: 调整结构但不改变预期行为
- `测试`: 补充或调整测试
- `文档`: 更新说明、接口文档、使用指南
- `构建`: 调整构建、脚本、依赖或配置
- `清理`: 删除废弃代码或无效资源

## 推荐写法

- 使用动宾结构，例如“补充 shell 扩展分组逻辑”。
- 把范围写清楚，例如模块名、子系统名、功能名。
- 保持首行聚焦一个动作，不在标题里塞多个并列结果。

## 示例

- `重构: 按功能拆分 shell 增强扩展`
- `修复: 纠正 PCI 寄存器位定义`
- `测试: 补充 qemu 本地回归脚本`
- `文档: 完善用户态构建步骤说明`
- `构建: 调整 aarch64 工具链配置`

## 正文模板

只有在首行无法说清楚时再写正文，可使用下面模板：

```text
原因:
- ...

主要改动:
- ...

验证:
- ...
```

正文同样使用中文，保持事实描述，不写空话。
